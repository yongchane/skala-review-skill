# 플랫폼별 출력 형식

## Velog

제목, 짧은 도입, 내용에 맞는 소제목, 사용자가 제공한 코드 블록을 포함한 Markdown을 바로 복사할 수 있게 제공한다. 이미지 위치를 요청하면 `[이미지: 설명]` 자리표시자를 넣는다.

## Notion

Notion에 붙여 넣기 쉬운 학습 노트로 제공한다. 제목, 핵심 정리, 개념 설명, 복습 문답 또는 실습 기록, 다음 복습 질문을 포함한다. Notion API 전용 블록을 요구하지 않는다.

## Tistory

검색하기 쉬운 제목, 짧은 도입, 읽기 좋은 소제목, 마무리 정리를 갖춘 블로그형 Markdown으로 제공한다. 사용자가 명시적으로 요청한 경우에만 HTML을 제공한다.

## 공통 규칙

- 기본 결과는 대화창에서 바로 복사할 수 있는 형태로 제공한다.
- 긴 완성 글부터 만들지 않는다. 먼저 추천 제목, 3~5개의 구성 항목, 항목별 2~4문장의 짧은 초안, 추가·확인할 내용을 제공한다.
- 복습 글 기본 초안은 약 700~1,200자를 권장한다. 사용자가 자세한 글을 요청할 때만 분량을 확장한다.
- 가능한 경우 사용자의 질문, 답변, 오개념 또는 수정된 이해를 반영한다.
- 빈 내용을 임의로 채우지 말고 `확인 필요`처럼 불확실성을 표시한다.
- 플랫폼 게시가 완료됐다고 말하지 않는다. 실제 게시 작업은 사용자가 수행한다.

## 플랫폼별 복습 글 작성 가이드

완성 초안이 아니라 작성 방법을 요청받으면 다음 순서로 제공한다.

1. 선택한 플랫폼과 예상 독자
2. 입력 내용에 어울리는 글 유형과 선택 이유
3. 추천 제목과 목차 3~5개
4. 각 항목에 넣을 내용과 작성 순서
5. 질문·오개념·예시·코드·이미지를 배치할 위치
6. 플랫폼에 맞는 말투와 표현 방식
7. 글을 작성하기 전에 추가로 확인할 내용

- 사용자의 메모와 실제 복습 대화에서 어떤 내용을 활용했는지 설명한다.
- 가이드는 완성 글을 대신 쓰지 않고 구성과 작성 방향을 중심으로 간결하게 제공한다.
- 각 항목의 시작 문장 예시는 짧게 제공할 수 있다.
- 사용자가 초안을 요청하면 선택한 가이드를 바탕으로 별도의 복습 글 초안을 작성한다.
- 제공되지 않은 수업 내용, 출처, 실행 결과를 만들지 않는다.
