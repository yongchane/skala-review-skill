# 웹 브라우저에서 사용하기

터미널이나 코딩 도구가 없어도 ChatGPT, Gemini, Claude 웹에서 SKALA 복습 흐름을 사용할 수 있습니다. 웹 버전은 프로그램을 설치하는 대신 README의 환경 라우터와 자기완결형 웹 실행 번들을 현재 대화에 적용합니다.

## README 링크로 바로 시작하기

저장소 루트나 `.git` clone 주소가 아니라 README 파일 직접 링크를 사용합니다. 저장소 루트 링크만 전달하면 일부 웹 AI는 README를 읽기 전에 접근에 실패하거나 이 저장소를 코드 리뷰 Skill로 잘못 추측할 수 있습니다.

```text
https://github.com/yongchane/skala-review-skill/blob/main/README.md
이 README를 직접 열고 최상단의 AI 실행 라우터에 따라 진행해.
저장소를 추측하거나 코드 리뷰 설명을 하지 마.
```

웹 AI가 README를 읽을 수 있다면 README의 AI 실행 라우터에 따라 [자기완결형 웹 실행 번들](SKALA-REVIEW-WEB-BUNDLE.md)로 이동해 설치 불가 안내나 코드 리뷰 설명을 하지 않고 첫 복습 질문부터 시작합니다.

GitHub 접근 가능 여부는 서비스, 계정, 대화의 탐색 기능에 따라 달라집니다. README 직접 링크 방식은 ChatGPT 웹에서 첫 복습 질문까지 확인했지만, 모든 웹 서비스에서 링크 열기를 보장하지는 않습니다. GitHub 내용을 읽지 못했다면 같은 요청을 반복하지 말고 웹 실행 번들을 첨부합니다.

## 가장 빠르게 시험하기

1. [자기완결형 웹 실행 번들](SKALA-REVIEW-WEB-BUNDLE.md)을 엽니다.
2. 파일 전체를 복사하거나 `.md` 파일로 내려받습니다.
3. 새 웹 대화에 전체 내용을 붙여 넣거나 내려받은 파일을 첨부합니다.
4. 다음 문구를 그대로 입력합니다.

```text
첨부하거나 붙여 넣은 SKALA 웹 실행 번들을 이 대화에 적용해줘.
지침을 요약하거나 설명하지 말고 바로 SKALA 복습을 시작해줘.
```

정상적으로 적용되면 AI는 설치 방법을 안내하는 대신 `오늘 무엇을 배웠나요?`라고 물어봅니다.

저장소 링크 방식이 실패하면 같은 요청을 반복하지 말고 이 파일 첨부 방식으로 전환합니다.

## 준비할 파일

- [자기완결형 웹 실행 번들](SKALA-REVIEW-WEB-BUNDLE.md): 일반 대화 첨부와 전체 기능 실행에 사용합니다.
- [간소화된 공통 복습 지침](SKALA-REVIEW-WEB-INSTRUCTIONS.md): 맞춤 기능의 글자 수 제한이 있을 때 Instructions에 붙여 넣습니다.
- `knowledge/`: 서비스가 지식 파일 업로드를 지원할 때 선택적으로 추가합니다.
  - `skala-curriculum-map.md`: 공개된 큰 교육 과정과 학습 연결
  - `output-profiles.md`: Velog·Notion·Tistory 출력 기준
  - `note-schema.md`: 나중에 다시 복습할 Markdown 노트 구조

지식 파일을 올리지 않아도 공통 복습 지침만으로 기본 흐름을 사용할 수 있습니다.

## 적용 범위와 유지 기간

웹에서 `이 스킬을 적용해줘`라고 요청하는 것만으로 계정에 영구 설치되지는 않습니다.

| 적용 방식 | 유지 범위 |
|---|---|
| 일반 채팅에 지침 전체 전달 | 현재 채팅에서만 적용 |
| ChatGPT Project instructions | 해당 Project 안의 채팅 |
| 맞춤 GPT | 해당 GPT로 시작한 채팅 |
| Gemini Gem | 해당 Gem으로 시작한 채팅 |
| Claude Project 또는 웹 Skill | 해당 Project·Skill을 사용하는 채팅 |
| Codex·Claude Code Skill 설치 | 로컬에서 제거하기 전까지 재사용 |

일반 채팅 방식은 빠른 체험용입니다. 계속 사용할 경우 Project·맞춤 GPT·Gem에 등록하는 편이 편리하지만, 이 설정도 계정 전체가 아니라 해당 공간에서만 적용됩니다.

GitHub의 공통 복습 지침이 업데이트되어도 이미 등록한 설정은 자동으로 바뀌지 않습니다. 새 버전을 사용하려면 [공통 복습 지침](SKALA-REVIEW-WEB-INSTRUCTIONS.md)을 다시 복사해 기존 지침을 교체합니다.

## ChatGPT 웹

### 맞춤 GPT를 만들 수 있는 경우

1. 새 맞춤 GPT를 만듭니다.
2. 이름을 `SKALA 복습 도우미`로 정합니다.
3. [간소화된 공통 복습 지침](SKALA-REVIEW-WEB-INSTRUCTIONS.md)의 전체 내용을 Instructions에 붙여 넣습니다.
4. [웹 실행 번들](SKALA-REVIEW-WEB-BUNDLE.md)을 지식 자료로 추가합니다.
5. 시작 문구에 `SKALA 복습 시작`을 추가하고 저장합니다.

### Project를 사용하는 경우

1. SKALA 복습용 Project를 만듭니다.
2. Project instructions에 간소화된 공통 복습 지침을 붙여 넣습니다.
3. 웹 실행 번들을 Project 파일로 추가합니다.
4. 같은 Project 안에서 새 대화를 열고 `SKALA 복습 시작`이라고 입력합니다.

## Gemini 웹

1. 새 Gem을 만듭니다.
2. 이름을 `SKALA 복습 도우미`로 정합니다.
3. Gem의 Instructions에 간소화된 공통 복습 지침 전체를 붙여 넣습니다.
4. 현재 계정에서 파일 추가를 지원하면 웹 실행 번들을 추가합니다.
5. 저장 후 `SKALA 복습 시작`이라고 입력합니다.

## Claude 웹

### Project를 사용하는 경우

1. SKALA 복습용 Project를 만듭니다.
2. Project instructions에 간소화된 공통 복습 지침 전체를 붙여 넣습니다.
3. 웹 실행 번들을 Project knowledge에 추가합니다.
4. 새 대화에서 `SKALA 복습 시작`이라고 입력합니다.

### 웹 Skill 기능을 사용할 수 있는 경우

웹 Skill 작성 화면의 지침에 간소화된 공통 복습 지침을 넣고 웹 실행 번들을 참고 자료로 추가합니다. 기능 제공 여부와 메뉴 이름은 계정 또는 서비스 업데이트에 따라 달라질 수 있으므로, 보이지 않으면 Project 방식을 사용하세요.

## 맞춤 기능을 만들 수 없는 경우

일반 대화의 첫 메시지에 웹 실행 번들 전체를 붙여 넣거나 파일로 첨부한 뒤, 이어서 다음처럼 입력합니다.

```text
첨부하거나 붙여 넣은 SKALA 웹 실행 번들을 이 대화에 적용해줘.
지침을 요약하거나 설명하지 말고 바로 SKALA 복습을 시작해줘.
```

이 방식은 현재 대화에서만 유지될 수 있으므로, 다음에 복습할 때는 지침을 다시 붙여 넣어야 할 수 있습니다.

## 웹에서 복습 노트 다시 보기

웹 AI가 다른 대화의 로컬 파일을 자동으로 기억한다고 가정하지 마세요. 복습이 끝나면 AI가 만든 Markdown 보관본을 직접 내려받거나 복사해 보관합니다.

다음 복습에서는 보관한 `.md` 파일을 첨부하거나 내용을 붙여 넣고 다음처럼 요청합니다.

```text
이전 복습 노트의 정답을 바로 보여주지 말고 먼저 회상 질문을 한 번에 하나씩 내줘.
```

## 자료 보호

- 제한된 강의 PDF나 슬라이드 전체를 올리지 않습니다.
- 본인이 작성한 키워드와 요약을 우선 사용합니다.
- 자료가 필요하면 공유 가능한 일부 페이지만 사용합니다.
- 개인정보, 계정 정보, API 키를 넣지 않습니다.
- 생성된 초안은 검토한 뒤 사용자가 직접 게시합니다.
